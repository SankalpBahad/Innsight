# CricInsight V0: Cricket Analytics Dashboard (Pandas-Based)

## Overview

CricInsight V0 is a high-performance cricket analytics dashboard built using a ball-by-ball IPL dataset. This version focuses on delivering fast, insightful analytics using a traditional data processing stack (Pandas + FastAPI), without a knowledge graph.

The goal of V0 is to provide a strong analytical foundation before introducing graph-based reasoning in future versions.

---

## Objectives

- Provide quick and interactive player analytics
- Enable matchup-based insights (player vs bowler, spin vs pace)
- Visualize scoring patterns and dismissal trends
- Build a scalable backend for large CSV datasets (~100MB)

---

## Tech Stack

### Backend
- FastAPI (Python)
- Pandas (data processing)

### Frontend
- React + Vite
- Tailwind CSS
- Recharts (charts)
- React Force Graph (basic relationships visualization)

---

## Dataset

Ball-by-ball dataset with fields including:

- Match details: p_match, date, year, competition
- Players: bat, bowl, team_bat, team_bowl
- Ball context: over, ball, phase-related info
- Outcome: runs, dismissal, extras
- Advanced: wagonX, wagonY, shot, control, wprob

Dataset size: ~99MB

---

## Architecture

Frontend (React)
      ↓
FastAPI Backend
      ↓
Pandas (In-memory Data Processing)

---

## Data Processing Strategy

- Load dataset once at startup
- Keep DataFrame in memory for fast access
- Precompute frequently used columns:
  - Strike rate
  - Phase (powerplay, middle, death)
  - Boundary flags (4s, 6s)

## Feature Breakdown

### 1. Career Overview

Provides:
- Total runs
- Batting average
- Strike rate
- Total balls faced
- Boundary counts (4s, 6s)

Implementation:
- Filter dataframe by batter
- Aggregate using groupby/sum/count

---

### 2. Matchup Analytics

Compare performance against:
- Pace vs Spin
- Individual bowlers

Metrics:
- Runs scored
- Balls faced
- Strike rate
- Dismissals

Implementation:
- Group by bowl_type or bowler name
- Compute aggregates

---

### 3. Scoring Zones (Wagon Wheel)

Visualizes where runs are scored using:
- wagonX, wagonY
- wagonZone

Output:
- Radar / polar chart

---

### 4. Dismissal Breakdown

Shows how a player gets out:
- Bowled
- Caught
- LBW
- Run out

Implementation:
- Count occurrences of dismissal types

---

### 5. Knowledge Graph (Basic - Non-KG)

NOTE: This is NOT a real graph DB.

Simulated relationships:
- Player → Bowlers faced
- Player → Venues played

Implementation:
- Aggregate top interactions
- Send as nodes + links JSON

---

### 6. Head-to-Head Table

Displays:
- Player vs Bowler stats
- Runs, balls, dismissals

## Backend Design

### Data Loading

- Load CSV at startup
- Store as global DataFrame
- Avoid repeated disk reads

---

### Processing Pattern

Typical flow:
1. Filter dataframe
2. Apply conditions
3. Aggregate results
4. Return JSON
